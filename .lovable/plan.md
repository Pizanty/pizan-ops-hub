# שדרוג תצוגת Dev Items

הבעיה בצילום: עמודת `OPEN` מכילה 76 פריטים בטור אחד אנכי בזמן ש-`IN PROGRESS` ריקה ו-`BLOCKED` כמעט ריקה. זה גורם לתחושה של "קיר טקסט" ולעבודה בלתי-אפשרית בעין.

## הפתרון — שלוש שכבות

### 1. סרגל סינון ומיון מעל הלוח
שורה דקה אופקית עם:
- חיפוש חופשי (`title` contains)
- צ'יפים לסינון לפי `type` (FEATURE / BUG / TASK / CHORE / MILESTONE) ולפי `priority` (P0..P3)
- toggle "רק שלי" (פריטים שיצרתי / מוקצים אליי)
- כפתור reset

הסינון משפיע על כל העמודות בו-זמנית, מצב נשמר ב-URL search params כדי שרענון לא יאפס.

### 2. עמודות צרות יותר + צפיפות גבוהה יותר בכרטיסים
- במסך רחב (`xl`) להציג את 5 העמודות אבל ברוחב שווה יותר ועם `min-w-0` כדי שלא תהיה עמודה ענקית אחת.
- כרטיס דחוס יותר: שורה אחת לכותרת (`line-clamp-2`), שורה אחת לתגיות (type · priority · ready), בלי תאריך אלא רק אם בתוך 7 ימים → מקטין את הגובה ב-~30%.

### 3. קיבוץ פנימי בעמודה לפי עדיפות + virtual "Show more"
זה הלב של התיקון. בתוך כל עמודה הפריטים מקובצים ב-3 sub-buckets לפי עדיפות:

```text
┌─── OPEN (76) ────────────┐
│  P0 · Critical    (2)   ▾│   ← פתוח כברירת מחדל
│   • card                 │
│   • card                 │
│                          │
│  P1 · High        (11)  ▾│   ← פתוח כברירת מחדל
│   • card ×11             │
│                          │
│  P2 · Medium      (38)  ▸│   ← מקופל, מציג 5 ראשונים + "Show 33 more"
│   • card ×5              │
│   [ Show 33 more ]       │
│                          │
│  P3 · Low         (25)  ▸│   ← מקופל לגמרי
└──────────────────────────┘
```

- ה-sub-buckets קולפסיביליים (כל אחד שומר state ב-`useState`).
- מתחת ל-bucket שיש בו יותר מ-5 פריטים מופיע "Show N more" שמרחיב inline (לא נווט לדף חדש).
- כשהעמודה ריקה לאחר סינון מציגים placeholder דק במקום עמודה ריקה ענקית.

### 4. תיקון רספונסיביות
- מובייל: גלילה אופקית snap בין עמודות (כמו היום).
- טאבלט (`md`): 2 עמודות + עמודה שלישית מתחת — לא 5 דחוסות.
- דסקטופ (`xl`): 5 עמודות, כל אחת `min-w-[16rem]` עם `min-w-0` בילדים כדי שטקסט יוכל להתקצר ב-`truncate` במקום לדחוף עמודה.

## פרטים טכניים

קובץ יחיד שמשתנה: `src/routes/dev.tsx`.

- **State חדש**: `filters` (search, types[], priorities[], onlyMine) ב-URL search params דרך `Route.useSearch()` + `navigate({ search })`.
- **חישוב**: `filteredItems = nonMilestoneItems.filter(matchesFilters)`, אחר כך `byStatus = group(filteredItems, 'status')`, אחר כך לכל status `byPriority = group(list, 'priority')`.
- **רכיב חדש פנימי**: `PriorityBucket` שמקבל `label`, `items`, `defaultOpen`, `previewCount=5`, מצייר כותרת קולפסיבילית + רשימת כרטיסים + כפתור "show N more".
- **רכיב חדש פנימי**: `DevCard` (מחילוץ הכרטיס הקיים) — כדי שאפשר יהיה לעטוף ב-`memo` ולא לרנדר מחדש את כל ה-76 כשמסננים.
- אין שינויים ב-API, אין מיגרציות, אין שינויים ב-`actions.server.ts`.

## מה לא נכלל בכוונה
- אין drag-and-drop בין עמודות (יש לזה ticket נפרד אם תרצה).
- אין שינוי לתצוגת ה-Milestones למעלה.
- אין שינוי לרכיב `DevSheet` (הטופס).

## תוצר
- עמודת OPEN נראית מנוהלת: 2 buckets פתוחים (P0+P1), 2 מקופלים (P2+P3).
- ניתן לסנן ל-"רק P1 שלי" בקליק אחד.
- הגובה הוויזואלי הכולל יורד מ-~76×80px ≈ 6000px ל-~1200px לפני הרחבה.
